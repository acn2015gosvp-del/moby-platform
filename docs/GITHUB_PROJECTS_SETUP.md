# 📊 GitHub Projects 설정 가이드

두 명이서 효율적으로 작업을 추적하기 위한 GitHub Projects 보드 설정 방법입니다.

## 🎯 보드 생성

1. GitHub 저장소에서 **Projects** 탭 클릭
2. **New project** 클릭
3. **Board** 템플릿 선택
4. 프로젝트 이름: `MOBY Development Board`

## 📋 컬럼 구성

다음과 같은 칸반 보드를 구성하세요:

```
📥 Backlog
  → 할 일 목록 (우선순위 낮음)

📋 To Do
  → 이번 주/이번 스프린트 작업

🔄 In Progress
  → 현재 작업 중인 항목

👀 Review
  → PR 생성 완료, 리뷰 대기 중

✅ Done
  → 완료된 작업
```

## 🏷️ 라벨 설정

다음 라벨들을 생성하세요:

### 우선순위
- `priority:high` (빨간색)
- `priority:medium` (노란색)
- `priority:low` (회색)

### 작업 유형
- `backend` (파란색)
- `frontend` (초록색)
- `bug` (빨간색)
- `feature` (보라색)
- `documentation` (회색)
- `testing` (주황색)

### 담당자
- `assignee:dev-a` (개발자 A)
- `assignee:dev-b` (개발자 B)

## 📝 이슈 생성 및 연결

### 이슈 생성 시
1. 적절한 템플릿 선택 (feature_request, bug_report, task_assignment)
2. 라벨 추가
3. 담당자 할당
4. Projects 보드에 자동 추가됨

### 이슈를 보드로 이동
1. 이슈에서 우측 사이드바의 **Projects** 클릭
2. 보드 선택
3. 컬럼 선택 (To Do, In Progress 등)

## 🔄 워크플로우

### 작업 시작
1. 이슈를 **To Do**에서 **In Progress**로 이동
2. 브랜치 생성 및 작업 시작

### PR 생성
1. 작업 완료 후 PR 생성
2. 이슈를 **In Progress**에서 **Review**로 이동
3. PR에 이슈 번호 링크 (예: `Closes #123`)

### 리뷰 완료
1. PR 승인 및 병합
2. 이슈를 **Review**에서 **Done**으로 이동
3. 이슈 자동으로 닫힘 (PR에 `Closes #123` 포함 시)

## 📊 주간 회고

매주 금요일:
1. **Done** 컬럼의 이슈들 확인
2. 완료된 작업 정리
3. 다음 주 계획 수립
4. **Done** 컬럼 정리 (보관용으로 별도 보드로 이동 가능)

## 💡 팁

- **작은 작업 단위**: 큰 작업은 여러 이슈로 분할
- **명확한 제목**: 이슈 제목만 봐도 무엇인지 알 수 있게
- **정기적 업데이트**: 하루 1-2회 보드 확인 및 업데이트
- **자동화**: GitHub Actions로 자동화 가능 (선택사항)
