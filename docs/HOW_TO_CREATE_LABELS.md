# 🏷️ GitHub 라벨 생성하기: 단계별 가이드

GitHub에서 라벨을 생성하는 방법을 단계별로 안내합니다.

---

## 📍 Step 1: Issues 탭으로 이동

1. GitHub 저장소로 이동: https://github.com/acn2015gosvp-del/moby-platform
2. 저장소 상단의 탭 메뉴에서 **Issues** 클릭

---

## 📍 Step 2: Labels 섹션 찾기

**Issues** 페이지에서:

1. 오른쪽 사이드바를 확인하세요
2. **Labels** 섹션을 찾으세요
3. **Labels** 옆에 있는 **⚙️ (톱니바퀴 아이콘)** 또는 **Labels** 텍스트를 클릭하세요

**또는**

1. **Issues** 페이지 상단의 **Labels** 버튼을 클릭하세요
   - 이 버튼은 이슈 목록 위쪽에 있습니다

---

## 📍 Step 3: New label 클릭

1. Labels 페이지로 이동하면
2. 오른쪽 상단에 **New label** 버튼이 있습니다
3. **New label** 버튼을 클릭하세요

---

## 📍 Step 4: 라벨 정보 입력

라벨 생성 폼이 나타나면 다음 정보를 입력하세요:

### 예시 1: 우선순위 라벨

**Label name:**
```
priority:high
```

**Description (선택사항):**
```
높은 우선순위 - 즉시 필요
```

**Color:**
- 색상 선택기를 클릭하거나
- 색상 코드를 직접 입력: `#d73a4a` (빨간색)

**Create label** 버튼 클릭

---

### 예시 2: 작업 유형 라벨

**Label name:**
```
backend
```

**Description:**
```
Backend 작업
```

**Color:**
- `#0e8a16` (초록색) 또는 원하는 색상

**Create label** 버튼 클릭

## ✅ 라벨 생성 확인

라벨을 생성한 후:

1. **Issues** 페이지로 돌아가기
2. 새 이슈를 생성할 때
3. **Labels** 섹션에서 생성한 라벨들이 보이는지 확인

---

## 🔧 기존 라벨 수정/삭제

### 라벨 수정
1. **Issues** → **Labels** 페이지로 이동
2. 수정할 라벨 옆의 **⚙️ (톱니바퀴 아이콘)** 클릭
3. 정보 수정 후 **Update label** 클릭

### 라벨 삭제
1. **Issues** → **Labels** 페이지로 이동
2. 삭제할 라벨 옆의 **⚙️** 클릭
3. 맨 아래 **Delete** 버튼 클릭
4. 확인 메시지에서 **Delete** 클릭

---

## 💡 팁

### 1. 라벨 이름 규칙
- 소문자 사용 권장
- 콜론(`:`)으로 카테고리 구분 (예: `priority:high`)
- 하이픈(`-`) 대신 언더스코어(`_`) 사용 가능

### 2. 색상 선택
- 같은 카테고리는 비슷한 색상 계열 사용
- 우선순위는 빨강-노랑-회색으로 구분
- 작업 유형은 각각 다른 색상 사용

### 3. 라벨 관리
- 너무 많은 라벨은 피하기
- 의미 있는 라벨만 생성
- 정기적으로 사용하지 않는 라벨 정리

---

## 🚀 빠른 생성 방법

모든 라벨을 한 번에 생성하려면:

1. **Issues** → **Labels** 페이지로 이동
2. **New label** 클릭
3. 위의 라벨 목록을 참고하여 하나씩 생성
4. 약 10-15분 정도 소요

---

**라벨 생성이 완료되면 이슈에 라벨을 추가할 수 있습니다!** 🏷️
